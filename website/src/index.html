Features

Everything you need to deploy.
Nothing you don't.

From AI-generated config to agent operations, bare-metal deploys, secure runtime isolation, and marketplace shipping, Stacker covers the full delivery path.

AI-Powered Configuration

Run stacker init --with-ai or stacker ai ask to scan your project, detect frameworks and services, update stacker.yml, and answer real deployment questions from your terminal.

Flagship Feature

Deep Project Scanning

Automatically detects Dockerfile, docker-compose.yml, package.json, Cargo.toml, requirements.txt, and the rest of the stack before it writes config.

Multi-Provider AI

Supports OpenAI, Anthropic, Ollama (local), and custom endpoints. Use local models for private infrastructure work or cloud models for fast iteration.

Cloud + Bare Metal Deploy

Deploy to DigitalOcean, Hetzner, AWS, Linode, UpCloud, or your own server with the same workflow.

Kata Runtime Isolation

Deploy with --runtime kata when you need hardware-isolated containers for sensitive workloads, while keeping runc as the fast default.

Agent Control + Firewall

Use stacker agent to inspect health, tail logs, restart apps, deploy directly, configure proxies, and manage iptables firewall rules on remote servers.

Pipes & Container Linking

Discover endpoints, map fields, trigger flows, replay executions, and connect apps together with stacker pipe automation.

Marketplace Shipping

Package and submit your stack, check review status, and follow feedback with stacker submit and stacker marketplace.

Service Catalog

20+ built-in service templates — Postgres, Redis, WordPress, Elasticsearch, and more. Add with stacker service add or let AI update the stack for you.

SSH Key Management

Generate, view, and upload SSH keys for your servers — all stored securely in HashiCorp Vault. No manual key juggling.

Latest Release

Stacker v0.2.9
brings smarter onboarding & deploy reliability.

The newest release adds AI config setup from the CLI, OAuth device auth, .env bootstrapping on deploy, improved Hetzner support, and hardened agent health and log workflows.

AI Setup

stacker config setup ai

Configure your AI provider, endpoint, model, and tasks directly from the CLI — Ollama-friendly with --provider, --endpoint, and --model flags.

Auth

OAuth device authorization

Browser-free device auth flow with default auth and API URLs stored in ~/.config/stacker/config.yml. stacker login now works in headless environments.

Deploy

.env bootstrapping

Cloud and server deploys automatically create missing .env files from adjacent .env.example files, with restrictive local permissions where supported.

Reliability

Preflight checks

New preflight validation catches config-bundle issues, rejects absolute destinations, and lists file mappings before sending deploy requests to avoid silent failures.

Networking

npm default network

Projects using npm now automatically get a default external network injected so services can communicate without manual compose network configuration.

Agent

Health, logs & npm detection

Agent health checks, log streaming, and npm project detection are all fixed. Separator widths are consistent. Hardened for live production environments.

Cloud

Hetzner location fixes

Location aliases like nbg1 are now normalized to installer-compatible datacenter values such as nbg1-dc3. Default server type updated to cx23.

Config

Config promote & inventory

Promote config keys across environments, diff inventories, check contracts, and use --remote to enrich targets with Vault-backed secret metadata without fetching values.

Secrets

Remote service secrets

Store Vault-backed secrets for deployable service and app targets from stacker.yml and supported Compose services, with metadata-only reads for safe inspection.

Operations

Cloud firewall controls

Open, remove, and list provider firewall rules without SSH, while guest OS firewall operations remain available through the agent for fine-grained control.

MCP

Remote secret tools

AI clients can list remote secret targets and manage service secret metadata through the MCP server while plaintext values stay safely in Vault.

Distribution

Marketplace workflows

Submit stacks for review with stacker submit, track approval status, and inspect reviewer feedback directly from the CLI with stacker marketplace.

Q&A

Common deployment questions.
Answered directly.

Agents

Can one Stacker project have more than one agent?

Yes. A single project can have multiple deployments, and each deployment runs its own Status Panel agent. That means staging and production can belong to the same project while still using different agents on different servers.

Targeting

How do I talk to the right agent?

Use --deployment <hash> when a project has more than one deployment. Without it, the CLI resolves the target from the local deployment lock or falls back to the most recently active deployment found from stacker.yml.
